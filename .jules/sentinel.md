## 2024-04-28 - SSRF in Image Fetching for Gemini
**Vulnerability:** A Server-Side Request Forgery (SSRF) vulnerability existed in `src/app/actions/tag-image.ts`. The server actions `generateImageTags` and `generateProductDescription` accepted arbitrary image URLs from the client and fetched them without validating the domain. This could allow an attacker to make the server perform requests to internal or unauthorized external resources.
**Learning:** Even when fetching images to process with external APIs (like Gemini), input URLs must be validated against a strict allowlist of trusted domains (e.g., the project's own Supabase storage) to prevent the server from acting as a proxy for malicious requests.
**Prevention:** Implement a strict URL validation helper (`validateImageUrl`) that checks the domain of incoming URLs against trusted environment variables (like `NEXT_PUBLIC_SUPABASE_URL`) before executing the `fetch` request. Ensure the validation fails securely (throws an error) if the environment variable is missing.
